# Review Checklist — 人工审核清单（打印版）

## 使用方式

每次运行 `python run.py daily` 后，打开 `data/reports/review_queue_YYYY-MM-DD.md`，
对每个需要审核的候选逐项确认以下 8 项。

## 8 项审核清单

### [ ] 1. 选题是否符合账号定位？
- AI布道账号：是否涉及 AI/ML/LLM？
- 工具拆解账号：是否有可运行代码/demo？
- 出海GEO账号：是否有跨境/国际化场景？

### [ ] 2. README 是否清晰说明用途和核心功能？
- 有中文 README → 加分
- 只有代码无文档 → 需补充信息
- 有 demo/gif/screenshot → 加分

### [ ] 3. 许可证是否允许内容引用？
- MIT / Apache 2.0 / CC → 可以
- GPL → 可以引用但注意
- 无许可证 → 需确认原作者意图

### [ ] 4. Star 数是否在合理范围？
- < 100 stars → 需人工确认质量（可能是新项目）
- 100-5000 → 正常范围
- > 50000 → 可能是过气热点

### [ ] 5. 最近一周是否有 commit 或 release？
- 最近 7 天有更新 → 活跃
- 1 个月内 → 正常
- > 6 个月 → 可能已停止维护

### [ ] 6. 竞品账号是否已经做过类似内容？
- 搜索关键词确认
- 如果已有人做过 → 需要差异化角度

### [ ] 7. 项目是否涉及敏感内容？
- NSFW / adult / deepfake → 一票否决
- 隐私 / 监控 / 键盘记录 → 一票否决
- 政治敏感 → 一票否决

### [ ] 8. 是否值得分配 30-60 分钟制作？
- 内容复杂度适中
- 有足够素材 (README/demo/docs)
- 预计能产生有吸引力的内容

## 审核决策速查

| 情况 | 决策 |
|------|------|
| 8/8 通过 | 采用，直接生成内容包 |
| 5-7/8 通过 | 补充信息后采用 |
| 3-4/8 通过 | 降级为常青候选，择日再审 |
| 1-2/8 通过 | 拒绝，不纳入选题库 |
